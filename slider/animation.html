当 DOM 元素的 animation 属性出现了一个新的 name，浏览器会播放这个动画；当 DOM 元素的 display 从 none 到其他属性（DOM 元素被绘制在页面中），浏览器会播放 DOM 元素上所有的 animation 动画（visibility 无用）

性能优化

浏览器渲染的关键工作

  • HTML 解析（Parse HTML）
  • 解析 CSS（Parse CSS）
  • 生成渲染树（Render Tree / Frame Tree）
  • 排版 / 重排（Layout / Reflow）
  • 绘图 / 重绘（Painting）

触发重排的属性

  • 盒子模型相关属性，如 width, height, margin, padding, border 等
  • 定位属性，如 left, top 等
  • 文字属性，如 font-size，font-family, text-align

触发重绘的属性

color, border-style, background, box-shadow

复合层（Composite Layers）不触发重排和重绘

使用 transform 和 opacity 变换都将成为复合层

一些其他的性能优化

  • 使用 requestAnimationFrame 方法，这个方法每 1 / 60 秒执行一次，以保证网站的动画帧数至少有 60 帧
  • 图片，视频资源的压缩和懒加载
  • 优化 JS 代码，减少不必要的开销
